Database Storage Engine

This project is required to be private by word of the instructor. If you'd like to view the source code, please e-mail me directly at rsg23@sfu.ca or rajangrewal123@gmail.com.

The premise of this project was to build a database management system. It was completed over the period of three months in C++ with a partner for a databases course taken at SFU. The base idea of the project is to manage database tables (denoted as pages), which includes modification, removal, and addition of tables, and records within tables. A basic benchmarking system displays the speeds Some features of this project include the following:

File Abstraction: contains basic definition and implementation of a page.
Buffer Manager: manages tables within the database.
Skip List: efficient data structure used to store records.
Lock Manager: structure of locks which can be acquired by transactions, allowing for concurrency.
Log: keeps records of all transactions containing how and when it modified the database.
User Functions: ability to insert, read, delete, and update specified records.
Benchmarking: contains statistics on the speed of the engine, primarily measure by commits and aborts per second.

Skills utilized

Databases, C++, concurrency, data structures, Git, collaboration.

Embedded SQL Interface

Github link

This project involved a fusion of front and back end technologies. Though the prime focus was databases, an interface was still required to be developed. It was completed with one other member over about two weeks. The program was designed using JavaScript and PostgreSQL. The way that this program works is that it displays options to the user regarding information and operations they would like to receive or perform on a database of grant proposals, reviewers, and meetings.

Database design improvements: a database was given, but it lacked some information, and contained errors. This was fixed through database definition modification.
Data insertion: information was required to be inserted into the database, which also aided testing.
Queries: queries were written in PostgreSQL which would perform numerous operations on the database.
Interface: Node.js was used to create the command-line interface.
Front & back-end fusion: allowed the front-end to interact with the back-end through JavaScript packages "Prompts" and "Clients".
Instructions & presentation: outlined how to set the program up and described how to use it in a report.

Skills utilized

PostgreSQL, JavaScript, Node.js, databases, Git, collaboration, writing.
