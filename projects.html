Amazon Web Scraper

Github link

This program allows the user to know when an Amazon item goes on sale. The way it works is that it requires an Amazon item link, a user agent, two email addresses, a sender and receiver, and a price. The program will then email the receiver's address when the specified item reaches or drops below the inputted price. Detailed instructions are provided within the scraper.py file in the GitHub repo link.

Scraping: uses BeautifulSoup to parse information from Amazon.
Emailing: uses SMTPLib to automatically send emails to specified addresses with a customizable message.
Price checking: checks the price by sending/receiving HTTP requests with the Requests library.
Refreshing: allows the user to enter a number which would represent the seconds between each price check.

Skills utilized

Python, BeautifulSoup, Requests, SMPTLib, HTTP, web scraping, HTML.

Multithreaded Calculator

Github link

This program was designed in C over the course of two weeks. It supports addition and multiplication operations and can also understand inputs with brackets in them. Each operation runs on its own thread, plus a thread for reading and formatting. The program features many keystone operating system principles which are outlined below.

Operations: supports addition, multiplication, and bracket usage.
Multithreading: increases program speed by having each operation run on its own thread.
Mutual exclusion & semaphores: locks shared structures and variables using semaphores. Deadlock avoidance: checks for malformed inputs which may cause deadlock and exits gracefully.
Race condition avoidance: avoids race conditions through appropraite mutual exclusion.
Critical sections: minimizies critical sections without giving way for a race condition.

Skills utilized

C, concurrency, mutual exclusion, deadlocking.

Sudoku Solver

Github link

The goal of this project was to solve sudoku instances as efficiently as possible. I coded this over about two weeks in Python. The algorithm used was a popular logical algorithm called DPLL. The algorithm uses a few heuristics to speed up the solving process which resulted in a tremendous difference. The algorithm will either print 'SAT' when it solves the sudoku instance, or 'UNSAT' if the puzzle is not solvable. Each sudoku puzzle is represented by a set of clauses which are in conjuctive normal form.

Recursive elimination: recursively searches the list of clauses given and eliminates satisfiable clauses by assigning values to variables.
Pure literal elimination heuristic: a pure literal is a variable which occurs exclusively positively or negatively. These variables can be used to speed up the search and this is exactly what this heuristic does.
Unit clause heuristic: a unit clause is a clause which contains one variable. This variable must be set so that this clause is true, so we can set other instances of this variable to this value and speed up the search significantly.

Skills utilized

Python, artificial intelligence, logic.
